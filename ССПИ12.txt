Во время трассировки маршрута(trace(Cisco IOS) / tracert(Windows)) происходит отправление 3 Echo Request с ttl = 1, 
в следствие чего ближайший роутер отправляет TTL expired запросившему. Затем снова отпаравляется 3 ER с ttl += 1, ttl = 2.
И так далее, пока не доберётся до целевого адреса.

tracert в Windows построен сугубо на ICMP, то есть на Echo Request и Echo Reply, в Linux - подобных операционных системах используется UDP для tracert
